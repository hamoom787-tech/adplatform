# SKILLS — منصة ربح من مشاهدة الإعلانات

## 1) فكرة المشروع
منصة ويب تسمح للمستخدمين بتسجيل الدخول، مشاهدة إعلانات معروضة داخل الموقع بشكل قانوني ومتوافق مع سياسات الإعلانات، ثم تجميع رصيد/نقاط يمكن استخدامها داخل المنصة حسب قواعد واضحة وشفافة.

> **مهم:** يجب الالتزام بسياسات Google AdSense/Google Ads وعدم تشجيع النقرات غير الحقيقية أو أي سلوك مخالف. الهدف هنا هو بناء تجربة مستخدم وتنفيذ تقني نظيف ومتوافق.

## 2) الهدف من المشروع
- إنشاء واجهة سهلة الاستخدام ومناسبة للموبايل والكمبيوتر.
- عرض الإعلانات داخل أماكن واضحة ومنظمة.
- إظهار رصيد المستخدم، عدد المشاهدات، وسجل الأرباح.
- بناء لوحة تحكم بسيطة للإدارة.
- تجهيز المشروع ليكون أساسًا قابلًا للتطوير لاحقًا.

## 3) المزايا الأساسية
- تسجيل/دخول المستخدم.
- صفحة رئيسية تعرض الرصيد والإحصائيات.
- صفحة مشاهدة الإعلانات.
- عداد وقت أو تقدم لكل إعلان.
- نظام نقاط/رصيد بعد إكمال المشاهدة.
- سجل العمليات: مشاهدة، ربح، سحب.
- صفحة سحب الأرباح.
- لوحة تحكم للمشرف.
- تصميم متجاوب Responsive.

## 4) صفحات المشروع المقترحة
1. **Home** — الصفحة الرئيسية.
2. **Login / Register** — تسجيل الدخول وإنشاء حساب.
3. **Dashboard** — الرصيد، الإحصائيات، آخر العمليات.
4. **Watch Ads** — عرض الإعلانات مع عداد.
5. **Withdraw** — طلب سحب الأرباح.
6. **Admin Panel** — إدارة المستخدمين والإعلانات والطلبات.

## 5) تقسيم الأدوار
### المستخدم
- يدخل للحساب.
- يشاهد الإعلانات.
- يجمع نقاط أو رصيد.
- يطلب سحب الأرباح.

### الأدمن
- يضيف/يحذف الإعلانات.
- يحدد قيمة الربح لكل مشاهدة.
- يراجع طلبات السحب.
- يراقب النشاط المشبوه.

## 6) قواعد العمل المقترحة
- كل إعلان له مدة مشاهدة محددة.
- لا يتم احتساب الربح إلا بعد اكتمال الوقت.
- يوجد حد أقصى يومي للمشاهدات.
- يتم تسجيل كل عملية في قاعدة البيانات.
- أي نشاط غير طبيعي يتم مراجعته يدويًا.

## 7) المتطلبات التقنية
- **Frontend:** HTML, CSS, JavaScript.
- **Backend (اختياري لاحقًا):** Node.js / PHP / Python.
- **Database (اختياري لاحقًا):** MySQL / MongoDB / Firebase.

## 8) شكل الواجهة
- ألوان حديثة وهادئة.
- بطاقات (Cards) لعرض الرصيد والإحصائيات.
- زر واضح لبدء مشاهدة الإعلان.
- شريط تقدم أثناء المشاهدة.
- تصميم بسيط وسريع التحميل.
- دعم كامل للموبايل.

## 9) Prompt محسّن لتوليد المشروع
استخدم هذا البرومبت مع أي AI لتوليد واجهة أو مشروع أولي:

**Prompt:**
> أنشئ لي مشروع ويب كامل بواجهة حديثة لمنصة ربح من مشاهدة الإعلانات، باستخدام HTML وCSS وJavaScript فقط في ملف واحد أو ملفات منفصلة حسب الأفضل.  
> المطلوب واجهة عربية RTL، متجاوبة مع الموبايل، تشمل: صفحة رئيسية، لوحة تحكم للمستخدم، صفحة مشاهدة إعلان مع عداد وقت، عرض الرصيد والنقاط، سجل الأرباح، وصفحة سحب الأرباح.  
> استخدم تصميمًا احترافيًا يشبه التطبيقات المالية الحديثة: بطاقات أنيقة، أزرار واضحة، ألوان متناسقة، وأيقونات مناسبة.  
> أضف تفاعلات JavaScript مثل: تحديث الرصيد، العد التنازلي، شريط تقدم، وإظهار حالة اكتمال المشاهدة.  
> اجعل الأكواد منظمة، نظيفة، وسهلة التعديل، مع تعليقات داخل الكود.  
> لا تستخدم أي مكتبات خارجية إلا إذا كانت ضرورية جدًا.  
> مهم: اجعل المشروع قابلًا للتطوير لاحقًا وربطه بباك إند وقاعدة بيانات.

## 10) Prompt أقوى لواجهة احترافية
**Prompt احترافي مختصر:**
> صمّم واجهة عربية احترافية لمنصة إعلانات ونقاط باستخدام HTML, CSS, JavaScript فقط.  
> أريد Dashboard حديثة فيها: رصيد المستخدم، عدد المشاهدات اليوم، الأرباح الإجمالية، زر مشاهدة إعلان، عداد تنازلي، وسجل آخر العمليات.  
> اجعل التصميم Responsive، سريع، ومناسب للموبايل، مع تأثيرات بسيطة واحترافية، وتنظيم ممتاز للأكواد.

## 11) ملاحظات مهمة
- لا تبنِ أي جزء يعتمد على نقرات وهمية أو تحايل على نظام الإعلانات.
- استخدم عرض الإعلانات بالطريقة المسموح بها من المنصة الإعلانية.
- افصل بين منطق الواجهة والمنطق الخلفي لاحقًا.
- اجعل كل شيء قابل للتعديل بسهولة.

## 12) نسخة جاهزة للنسخ السريع
**عنوان المشروع:** منصة ربح من مشاهدة الإعلانات  
**التقنيات:** HTML / CSS / JavaScript  
**الهدف:** واجهة عربية حديثة تعرض الإعلانات، الرصيد، المشاهدات، وسحب الأرباح  
**الأسلوب:** احترافي، متجاوب، بسيط، ومناسب للتطوير لاحقًا
